# ms-account

Hexabank — Account microservice

## Overview

`ms-account` implements account and movement management for Hexabank. It provides REST endpoints to create accounts, register movements (deposits/withdrawals), and generate account statements in JSON and Excel formats. The service uses Spring Boot, Spring Data JPA, Flyway for schema migrations, and Micrometer/Prometheus for metrics.

## Architecture

- Java 21, Spring Boot 4
- Layered architecture: domain, application (use cases), infra (controllers, persistence, kafka adapters)
- Persistence: PostgreSQL (migrations with Flyway)
- Messaging: Spring Kafka (producer for account events)
- Observability: Actuator endpoints (health, metrics, prometheus) and MDC logging

## How to run locally (development)

Prerequisites:

- JDK 21
- Gradle (optional, project includes wrapper)
- Docker & Docker Compose (recommended)

Run application with Gradle (dev profile uses H2 in-memory DB):

```bash
cd ms-account
./gradlew bootRun
```

Run tests:

```bash
cd ms-account
./gradlew test
```

## Run with Docker Compose (recommended for full stack)

From repository root:

```bash
docker compose up --build
```

This brings up Postgres, Kafka (local images), ms-account and ms-client services. Environment variables and ports are configured in `docker-compose.yml` at the repository root.

## Database migrations and seed data

Flyway migrations are in `src/main/resources/db/migration`. Migrations V1..V5 are applied on startup. V5 seeds sample accounts and movements used by acceptance tests and demos.

If you need a consolidated SQL file, a `BaseDatos.sql` can be generated by concatenating migrations.

## Endpoints

Base path: `/`

- POST /accounts
  - Create a new account. JSON body example:
    {
    "clientId": 1,
    "accountNumber": "478758",
    "accountType": "Ahorros",
    "ownerName": "Jose Lema",
    "initialBalance": 2000
    }
- GET /accounts
  - List accounts
- GET /accounts/{id}
  - Get account by internal id
- POST /accounts/{accountNumber}/movements
  - Register deposit/withdrawal for an account. Body example:
    {
    "movementType": "DEPOSIT",
    "amount": 600
    }
- GET /accounts/{accountNumber}/movements?from={yyyy-MM-dd}&to={yyyy-MM-dd}
  - List movements for account in date range
- GET /reports?clientId={id}&from={yyyy-MM-dd}&to={yyyy-MM-dd}
  - Generate JSON report for client
- GET /reports/excel?clientId={id}&from={yyyy-MM-dd}&to={yyyy-MM-dd}
  - Download Excel report

## Observability

- Actuator endpoints exposed: `/actuator/health`, `/actuator/info`, `/actuator/metrics`, `/actuator/prometheus`
- Prometheus scrape annotations are present in the Kubernetes manifest under `ms-account/k8s/deployment.yaml`.

## Postman collection

Import the collection `ms-account/docs/postman/Hexabank-ms-account.postman_collection.json` and environment `Hexabank-local.postman_environment.json` to manually validate endpoints. Adjust `baseUrl` if your service root differs (example: `http://localhost:8080/api`).

## Kubernetes

Kubernetes manifests are available in `ms-account/k8s` (Deployment + Service). They include readiness/liveness probes and Prometheus annotations.

## Notes and troubleshooting

- The database seed (V5) inserts sample accounts and movements used by acceptance tests; you can skip seeding by removing or editing V5 if necessary.
- If Kafka is not available, the application can be started with Kafka listeners disabled for local testing (application properties allow overriding `spring.kafka.bootstrap-servers`).
